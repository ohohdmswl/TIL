### 02. 라이브러리 살펴보기
- External Libaries 
	- 다운받은 라이브러리 내역 모두 확인 가능
	- 빌드 툴은 의존관계를 관리해줌(메이븐, 그래이들)
	- 그래이들 확인을 해보면 떙겨온 라이브러리 확인 가능하고 땡겨온 라이브러리가 필요로 하는 타 라이브러리를 의존관계 따져서 모두 땡겨옴
		- 참고로 (* ) 붙은 라이브러리는 위에서 이미 땡겨와서 중복을 제거했다는 표시
		- ![image](https://github.com/ohohdmswl/TIL/assets/132552661/db7aeacd-52a0-49dd-a958-a4d8e059c022)

* 로그 
	* slf4j VS logback
		* slf4j -> 인터페이스
		* logback -> 로그를 어떤 구현체로 출력할지 
	* 보통 스프링부트 라이브러리를 땡기면 자동으로 땡겨짐

* 테스트 라이브러리
	* junit -> 테스트 프레임워크
	* mockito -> 목 라이브러리
	* assertj -> 테스트 코드를 좀 더 쉽게 작성하게 도와주는 라이브러리
	* spring-test -> 스프링 통합 테스트 지원
